Lesson 7: general linear model

Readings: Kéry (2010) chapter 11

 

7.1 Introduction

The models we’ve encountered up to now (model of the mean, t-test, regression, ANOVA) are all special cases of the general linear model (not to be confused with the generalised linear model, which is still to come). The general model (also sometimes called ANCOVA) fits a continuous response variable as a function of discrete or continuous explanatory variables (often more than one of each, with interactions), plus random residual variation explained by a normal distribution.

For this lesson, we will return to simulating data based on snakes (Snakes? Did you say snakes?). Again we will look at length–mass relationships, and how they differ among three populations. Let’s imagine our three populations are at Madikwe (Mad), Rietvei (Rie), Sabi Sands (Sab) nature reserves. The means parametrisation of the model we will use is

\(y_i = \alpha_{j(i)} + \beta_{j(i)} \times x_i + \epsilon_i\)
\(\epsilon_i \sim {\sf Normal}(0, \sigma^2)\)

Interpretation:
\(y_i\): body mass of individual \(i\)
\(\alpha_{j(i)}\): intercept for population \(j\)
\(\beta_{j(i)}\): slope for population \(j\)
\(x_i\): body length of individual \(i\)
\(\epsilon_i\): describes the combined effects of unmeasured influences on body mass of snake \(i\)

 

The effects parametrisation is

\(y_i = \alpha_{Mad} + \beta_1 \times x_{Rei(i)} + \beta_2 \times x_{Sab(i)} + \beta_3 \times x_{body(i)} + \beta_4 \times x_{Rei(i)} \times x_{body(i)} + \beta_5 \times x_{Sab(i)} \times x_{body(i)}+ \epsilon_i\) \(\epsilon_i \sim {\sf Normal}(0, \sigma^2)\)

Interpretation:
\(y_i\) and \(\epsilon_i\): as before
\(\alpha_{Mad}\): expected mass of snakes at Madikwe
\(\beta_2\) and \(\beta_3\): differences in expected masses between Madikwe and Rietvlei or Sabi
\(\beta_4\) and \(\beta_5\): differences in slopes between Madikwe and Rietvlei or Sabi

 

7.2 Data simulation

# generate data for explanatory variables
n.groups <- 3
n.sample <- 10  
n <- n.groups * n.sample        # total no. data points
x <- rep(1:n.groups, rep(n.sample, n.groups)) # indicator for population
pop <- factor(x, labels = c("Madikwe", "Rietvlei", "Sabi"))
length <- runif(n, 45, 70)      # body lengths

# build design matrix and choose parameter values
Xmat <- model.matrix(~ pop*length)
# print(Xmat, dig = 2) # if you want to see it
beta.vec <- c(-250, 150, 200, 6, -3, -4)

# calculate linear predictor (expected values for mass)
# add random variation
lin.pred <- Xmat[,] %*% beta.vec    # value of lin.predictor
eps <- rnorm(n = n, mean = 0, sd = 10)  # residuals 
mass <- lin.pred + eps          # response = lin.pred + residual

# let's have a look
hist(mass)
snakes.df <- data.frame(length, mass, pop)
ggplot(data = snakes.df) +
  geom_point(aes(x = length, y = mass, colour = pop), size = 3) +
  labs(x = 'Body length (cm)', y = 'Body mass (g)')

In our data set, snakes from Madikwe have the steepest slope, followed by Rietvlei and then Sabi.

 

7.3 Classical analysis in R

It’s relatively simple to specify the effects parametrisation model in R. We can then compare them quickly to the values we set for the data simulation to see how we did.

summary(lm(mass ~ pop * length))

beta.vec
cat("And the residual SD was 10 \n")

 

7.4 Bayesian analysis with BUGS

In NIMBLE it’s easier to fit the means parametrisation of the model, and then use derived parameters to calculate the effects (differences) if we need them.

I should also include a cautionary note about using standardised covariates. Well have a look at that as we go through the examples, and I’ll return to that later in the lesson.

# specify the model
model.genlin1 <- nimbleCode({

# priors
  for (i in 1:n.group){
    alpha[i] ~ dnorm(0, sd = 30)        # intercepts
    beta[i] ~ dnorm(0, sd = 30)     # slopes
  }
  sigma ~ dunif(0, 100)         # residual standard deviation
  
  # likelihood
  for (i in 1:n){
    mass[i] ~ dnorm(mu[i], sd = sigma)
    mu[i] <- alpha[pop[i]] + beta[pop[i]]* length[i]
  }
  
  # derived quantities
  # define effects relative to baseline level
  a.effe2 <- alpha[2] - alpha[1]        # intercept Rietvlei vs Madikwe
  a.effe3 <- alpha[3] - alpha[1]        # intercept Sabi vs Madikwe
  b.effe2 <- beta[2] - beta[1]      # slope Rietvlei vs Madikwe
  b.effe3 <- beta[3] - beta[1]      # slope Sabi vs Madikwe
  
  # custom tests
  test1 <- beta[3] - beta[2]        # slope Sabi vs Rietvlei
})

# data and constants
my.data <- list(mass = as.numeric(mass), pop = as.numeric(pop), length = length)
my.consts <- list(n.group = max(as.numeric(pop)), n = n)

# initial values
my.inits <- function() list(alpha = rnorm(n.groups, 0, 2),
                            beta = rnorm(n.groups, 1, 1), sigma = rlnorm(1))

# parameters to estimate
my.params <- c('alpha', 'beta', 'sigma', 'a.effe2', 'a.effe3', 'b.effe2',
               'b.effe3', 'test1')

# MCMC settings
ni <- 1200
nb <- 200
nt <- 2
nc <- 3

# start Gibbs sampling
out1a <- nimbleMCMC(code = model.genlin1,
                    data = my.data,
                    constants = my.consts,
                    inits = my.inits,
                    monitors = my.params,
                    thin = nt, niter = ni, nburnin = nb, nchains = nc)

(outSum1a <- MCMCsummary(out1a, round = 2))         # look at output
beta.vec                # Truth in the data-generating process
summary(lm(mass ~ pop * length))    # the classical solution again

Wait a minute… the estimates are rather different between the classical and Bayesian analyses. What’s going on here? The small \(\hat{R}\) values indicate the Markov chains converged on an estimate, so it probably isn’t that. Remember that \({\tt alpha[1]}\) and \({\tt beta[1]}\) in NIMBLE correspond to the intercept and the length main effect in the analysis in R and \({\tt a.effe2}\), \({\tt a.effe3}\), \({\tt b.effe2}\), \({\tt b.effe3}\) to the remaining terms of the analysis in R. So \({\tt lm()}\) can produce parameter estimates similar to those used in generating the data, but not NIMBLE. Why is this?

The problem has to do with the scaling of the variable \(length\). In practice we want to standardise variables with a wide range of values so that the total spread is not too far from zero; otherwise, problems could arise. One such problem is convergence errors, but we would never suspect that looking at the values for \(\hat{R}\).

What’s the solution? One is to rerun the same model in a simpler form and see if we get the same output, or to compare to classical analysis. Alternatively we could plot the estimated lines on a graph with the data points. That would have demonstrated some problems.

Here we repeat the analysis with standardised values for \(length\).

# data and constants
my.data <- list(mass = as.numeric(mass), pop = as.numeric(pop),
                length = as.numeric(scale(length)))
my.consts <- list(n.group = max(as.numeric(pop)), n = n)

# initial values
my.inits <- function() list(alpha = rnorm(n.groups, 0, 2),
                            beta = rnorm(n.groups, 1, 1), sigma = rlnorm(1))

# parameters to estimate
my.params <- c('alpha', 'beta', 'sigma', 'a.effe2', 'a.effe3', 'b.effe2',
               'b.effe3', 'test1')

# MCMC settings
ni <- 1200
nb <- 200
nt <- 2
nc <- 3

# start Gibbs sampling
out1b <- nimbleMCMC(code = model.genlin1,
                    data = my.data,
                    constants = my.consts,
                    inits = my.inits,
                    monitors = my.params,
                    thin = nt, niter = ni, nburnin = nb, nchains = nc)

(outSum1b <- MCMCsummary(out1b, round = 2))         # look at output
beta.vec                # truth in the data-generating process

summary(lm(mass ~ pop * length))    # the classical solution again
print(lm(mass ~ pop * as.numeric(scale(length)))$coefficients, dig = 4)

So we see the analyses with the standardised values for length are consistent in their parameter estimates. So in the previous analysis in NIMBLE, the Markov chains did not converge, even though the values of \(\hat{R}\) said they did. As a lesson, we should always consider to transform covariate values, even if it makes interpretation and graphing more difficult later.

My preference is to normalise covariates like \(length\). That mean subtracting the mean and dividing by the standard deviation. Doing this would allow us to make the following interpretation: one standard deviation change in body length is associated with a \(\hat{\beta_1}\)-factor change in mass of snakes at Madikwe Nature Reserve. Sometimes centring alone is enough to solve the problem, in which case we could interpret relative to the original units of \(length\).
