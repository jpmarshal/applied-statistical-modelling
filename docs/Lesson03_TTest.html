Lesson 3: t-test

Readings: Kéry (2010) chapter 7

 

Most people wouldn’t think of the simple t-test as a linear model, but that is precisely what it is. In this lesson we will introduce ourselves to specifying linear models in BUGS using this starting point, and we will specify the “t-test model” for equal and unequal variances.

 

3.1 t-test with equal variances

3.1.1 Introduction

The model underlying the t-test with equal variances is:

\(y_i = \alpha + \beta \times x_i + \epsilon_i\)
\(\epsilon_i \sim {\sf Normal}(0, \sigma^2)\)

Interpretation:
\(y_i\): the response, a continuous variable measured for each unit \(i\) in the sample
\(x_i\): is an indicator variable for group 2
\(\alpha\): the mean for group 1
\(\beta\): the difference between groups 1 and 2
\(\epsilon_i\): the residuals around the group means
\(\sigma^2\): the variance of the distribution of the residuals

 

3.1.2 Data simulation

  • We will simulate data based on peregrine falcons that we encountered in an earlier lesson.
  • We’ll imagine that we have wingspan measurements for males and females, that they’re normally distributed, and we’ll assume that the mean and standard deviation are 77.5 and 2.5 cm for males and 105 and 3 cm for females.
n1 <- 60                    # Number of females
n2 <- 40                    # Number of males
mu1 <- 105                  # Population mean of females
mu2 <- 77.5                 # Population mean of males
sigma <- 2.75               # Average population SD of both
n <- n1 + n2                # Total sample size
y1 <- rnorm(n1, mu1, sigma) # Data for females
y2 <- rnorm(n2, mu2, sigma) # Date for males
y <- c(y1, y2)              # Aggregate both data sets
x <- rep(c(0,1), c(n1, n2)) # Indicator for male

ggplot() +
  geom_boxplot(aes(x = as.factor(x), y = y)) +
  xlab('Male') +
  ylab('Wingspan (cm)')

The method we used to generate these data correspond to a means parameterization. We could also generate a data set in a manner that demonstrates effects parameterization.

n <- n1 + n2                  # Total sample size
alpha <- mu1                  # Mean for females serves as the intercept
beta <- mu2 - mu1             # Beta is the difference male-female
E.y <- alpha + beta*x         # Expectation
y.obs <- rnorm(n = n, mean = E.y, sd = sigma) # Add random variation

ggplot() +
  geom_boxplot(aes(x = as.factor(x), y = y.obs)) +
  xlab('Male') +
  ylab('Wingspan (cm)')

Repeatedly executing this code chunk gives us a sense of the effect of chance (sampling variance or sampling error) on the resulting data, and the differences between repeated realizations of the same random process.

 

3.1.3 Classical analysis in R

To analyse our simulated data with a linear model in R:

fit1 <- lm(y ~ x)     # Analysis of first data set
fit2 <- lm(y.obs ~ x) # Analysis of second data set

summary(fit1)
summary(fit2)

The differences between the two analyses are the consequence of sampling variation. To get an ANOVA table for the two models:

anova(fit1)
anova(fit2)

We can also look at the design matrices:

model.matrix(fit1)
model.matrix(fit2)

 

3.1.4 Bayesian analysis with BUGS

Here is an analysis of the first data set using the Bayesian approach and NIMBLE:

library(nimble)

# specify the model
model.ttest1 <- nimbleCode({
  
  # Priors
  mu1 ~ dnorm(0, sd = 30)
  delta ~ dnorm(0, sd = 30)
  sigma ~ dunif(0, 100)
  
  # Likelihood
  for (i in 1:n) {
    y[i] ~ dnorm(mu[i], sd = sigma)
    mu[i] <- mu1 + delta*x[i]
    residual[i] <- y[i] - mu[i] # define residuals
  }
  
  # D4erived quantities: one of the greatest things about a Bayesian analysis
  mu2 <- mu1 + delta # Difference in wingspan
})

Next we must specify:

  • the data
  • the constants
  • initial values
  • parameters we want NIMBLE to remember for us
  • the Markov chain Monte Carlo (MCMC) settings

Then we send everything to NIMBLE.

# data and constants
my.data <- list(x = x, y = y)
my.consts <- list(n = n)

# initial values
my.inits <- function() list(mu1 = rnorm(1), delta = rnorm(1),
                            sigma = rlnorm(1))

# parameters to estimate
my.params <- c('mu1','mu2', 'delta', 'sigma', 'residual')

# MCMC settings
nc <- 3     # no. chains
ni <- 1000  # no. draws from posterior for each chain
nb <- 1     # no. draws to discard as burn in
nt <- 1     # thinning rate

# start Gibbs sampler
out1 <- nimbleMCMC(code = model.ttest1,
                   data = my.data,
                   constants = my.consts,
                   inits = my.inits,
                   monitors = my.params,
                   thin = nt, niter = ni, nburnin = nb, nchains = nc)

# save the summary for graphing
(outSum1 <- MCMCsummary(out1, round = 2))

Comparison of classical and Bayesian analyses:

  • Mean estimates are almost identical.
  • The residual standard deviation estimate (i.e. residual standard error in classical versus sigma in Bayesian) is slightly larger in the Bayesian analysis. This difference stems from the approximate asymptotic nature of frequentist inference compared to the exact inference under the Bayesian paradigm.
  • Very nice feature in Bayesian: derived parameters (mu2) that are functions of underlying parameters plus there uncertainties (mu1 and delta) can be obtained easily using the MCMC posterior samples. Unlike in classical analysis where the delta method would be needed, error from estimation is automatically propagated into functions of parameters.

Further comments:

  • The effective number of parameters to be estimated is three: one variance (sigma) and two means (mu1, mu2).
  • Before we use this or any model for inference we should check that it is adequate. Because we simulated data, the model should be adequate, but with real data we would want to a further assessment with the residuals.
  • For practice, we’ll assess model adequacy with our residuals. One observation is that each residual has its own posterior distribution, which might seem odd. But in Bayesian analysis, each calculated or estimated value (i.e. an unknown) has some degree of uncertainty represented by its posterior distribution.

We’ll plot the residual against the order of the individual in our data set, then we’ll assess the spread of residuals with box-plots to judge whether the spread is similar:

ggplot() +
  geom_point(aes(x = 1:100, y = outSum1$mean[4:103])) +
  geom_hline(yintercept = 0) +
  xlab('Individual') +
  ylab('Wingspan residuals (cm)')
ggplot() +
  geom_boxplot(aes(x = as.factor(x), y = outSum1$mean[4:103])) +
  geom_hline(yintercept = 0) +
  xlab('Male') +
  ylab('Wingspan residuals (cm)')

No pattern in the first graph, and the box-plots look similar. No evidence of violations.

 

3.2 t-test with unequal variances

3.2.1 Introduction

We’re now moving to the situation where variances are unequal between groups, something that is common in real data sets. We can modify the model from the equal-variance situation to accommodate unequal variance as follows:

\(y_i = \alpha + \beta \times x_i + \epsilon_i\)
\(\epsilon \sim {\sf Normal}(0, \sigma_1^2)\) when \(x_i = 0\) (females)
\(\epsilon \sim {\sf Normal}(0, \sigma_2^2)\) when \(x_i = 1\) (males)

3.2.2 Data simulation

Here is the code to simulate data under the heterogeneous group model:

n1 <- 60                # Number of females
n2 <- 40                # Number of males
mu1 <- 105              # Population mean for females
mu2 <- 77.5             # Population mean for males
sigma1 <- 3             # Population SD for females
sigma2 <- 2.5               # Population SD for males

n <- n1 + n2                # Total sample size
y1 <- rnorm(n1, mu1, sigma1)        # Data for females
y2 <- rnorm(n2, mu2, sigma2)        # Data for males
y <- c(y1, y2)              # Aggregate both data sets
x <- rep(c(0, 1), c(n1, n2))        # Indicator for male

ggplot() +
  geom_boxplot(aes(x = as.factor(x), y = y)) +
  xlab('Male') +
  ylab('Wingspan residuals (cm)')

3.2.3 Classical analysis in R

We can analyse data with heterogeneous variances in the classical paradigm using Welch’s test. This is the default when using the function \({\tt t.test}\):

t.test(y ~ x)

3.2.4 Bayesian analysis with BUGS

Here we specify the model using BUGS code and NIMBLE.

# specify the model
model.ttest2 <- nimbleCode({
  
  # Priors
  mu1 ~ dnorm(0, sd = 30)
  mu2 ~ dnorm(0, sd = 30)
  sigma1 ~ dunif(0, 1000)
  sigma2 ~ dunif(0, 1000)
  
  # Likelihood
  for (i in 1:n1){
    y1[i] ~ dnorm(mu1, sd = sigma1)
  }
  
  for (i in 1:n2){
    y2[i] ~ dnorm(mu2, sd = sigma2)
  }
  
  # Derived quantities
  delta <- mu2 - mu1
})

# data and constants
my.data <- list(y1 = y1, y2 = y2)
my.consts <- list(n1 = n1, n2 = n2)

# initial values
my.inits <- function() list(mu1 = rnorm(1), mu2 = rnorm(1),
                            sigma1 = rlnorm(1), sigma1 = rlnorm(1))

# parameters to estimate
my.params <- c('mu1','mu2', 'delta', 'sigma1', 'sigma2')

# MCMC settings
nc <- 3     # no. chains
ni <- 2000  # no. draws from posterior for each chain
nb <- 500   # no. draws to discard as burn in
nt <- 1     # thinning rate

# start Gibbs sampler
out2 <- nimbleMCMC(code = model.ttest2,
                  data = my.data,
                  constants = my.consts,
                  inits = my.inits,
                  monitors = my.params,
                  thin = nt, niter = ni, nburnin = nb, nchains = nc)

# save the summary for graphing
(outSum2 <- MCMCsummary(out2, round = 2))

Heterogeneous group variances are easy to deal with in a Bayesian framework.

  • If we wanted to formally test for a differences in variances, we could reparameterise to make one variance equal to the other variance plus a constant to be estimated.
  • This could be done within NIMBLE or in R with the saved Markov chain values for \(sigma1\) and \(sigma2\) (the latter is often easier).
  • If the posterior distribution covers zero, that could be taken as evidence for a lack of difference.

 

3.3 Summary

Although easy to conduct a t-test using the canned R function, building our own in BUGS language has helped us to learn some necessary lessons:

  • Bayesian modelling has flexibility to allow us to accommodate peculiarities of a particular data set (like heterogeneous group variances).
  • We can model not just the mean, but whatever parameter we need to (like sigma) to address a question or objective. We could even use covariates.
  • Modelling variance is relatively easy in the Bayesian paradigm, whereas in the classical paradigm it can be difficult for anyone but a hard-core statistician.
