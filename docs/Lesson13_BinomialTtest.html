Lesson 13: binomial t-test

Readings: Kéry (2010) chapter 17

 

13.1 Introduction

In this lesson, we shift from Poisson GLMs to binomial GLMs. They are useful for when we want to use count data to estimate binomial proportions, and the models we will use are commonly called logistic regression. The most important difference between Poisson and binomial counts is that the latter have some upper limit, and thus we are modelling bounded counts (the limit being trial size \(N\)). Because of this upper limit, we can express counts relative to that limit as well as proportions. Poisson counts, in contrast, lack a fixed upper limit.

We can think of a binomial random variable as modelling a series of coin tosses. From a total number of tosses (\(N\)) we might be interested in the number of heads, as well as the likelihood of getting a head on the next coin toss. That likelihood is the probability of a success: \(Pr(heads) = Pr(success) = p\). If we have \(r\) heads in a series of \(N\) tosses, then we write \(r \sim {\sf Binomial}(N, p)\). If we have a single toss (\(N = 1\)) then the associated distribution is called Bernoulli, having one parameter \(p\).

For this lesson, let’s imagine a botanist is exploring a number of nature reserves in the lowveld and is looking for two variants of the spotted aloe, Aloe greatheadii var. greatheadii and A. g. var. davyana. The botanist explores 50 sites and records whether she sees each variant or not, with the goal of assessing whether one variant is more restricted in its distribution than the other. For greatheadii she sees plants at 13 sites, and for davyana she sees plants at 29 sites. As we’ve mentioned in previous scenarios, we are assuming that plants of each variant are seen, if they are present, without error. Here is the model we will be fitting:

Distribution: \(C_i \sim {\sf Binomial}(N, p_i)\)
Link function: logit, \(logit(p) = \log \frac{p_i}{1 - p_i}\) = linear predictor
Linear predictor: \(\alpha + \beta \times x_i\)

Interpretation:
\(C_i\): the number of sites at which Aloe variant \(i\) is detected
\(x\): indicator variable for the davyana variant
\(\alpha\): logit-scale probability of occurrence of the greatheadii variant \(\beta\): difference in logit-scale probability of occurrence between greatheadii and davyana

 

13.2 Data simulation

Here we generate data from the example described in the Introduction. Our modelled response consists of two numbers.

# generate data
N <- 50                 # binomial total (no. coin tosses)
p.gr <- 13/50               # success probability greatheadii
p.da <- 29/50               # success probability davyana

# add binomial noise to each
C.gr <- rbinom(1, 50, prob = p.gr)   ;   C.gr
C.da <- rbinom(1, 50, prob = p.da)   ;   C.da
C <- c(C.gr, C.da)
variant <- factor(c(0,1), labels = c('greatheadii', 'davyana'))

 

13.3 Classical analysis in R

summary(glm(cbind(C, N - C) ~ variant, family = binomial))
predict(glm(cbind(C, N - C) ~ variant, family = binomial), type = 'response')

Our binomial t-test indicate strong evidence of a difference in numbers of each variant detected.

 

13.4 Bayesian analysis using BUGS

# specify the model
model.binomT <- nimbleCode({
  
  # priors
  alpha ~ dnorm(0, sd = 10)
  beta ~ dnorm(0, sd = 10)
  
  # likelihood
  for (i in 1:n){
    C[i] ~ dbin(p[i], N)        # note p before N
    logit(p[i]) <- alpha + beta *species[i]
  }
  
  # derived quantities
  Occ.great <- exp(alpha) / (1 + exp(alpha))
  Occ.davyana <- exp(alpha + beta) / (1 + exp(alpha + beta))
  Occ.Diff <- Occ.davyana - Occ.great # test quantity
})

# data and constants
my.data <- list(C = C)
my.consts <- list(N = N, species = c(0,1), n = length(C))

# initial values
my.inits <- function() list(alpha = rlnorm(1), beta = rlnorm(1))

# Parameters to estimate
my.params <- c('alpha', 'beta', 'Occ.great', 'Occ.davyana', 'Occ.Diff')

# MCMC settings
nc <- 3
ni <- 1200
nb <- 200
nt <- 2

# start Gibbs sampling
out1 <- nimbleMCMC(code = model.binomT,
                   data = my.data,
                   constants = my.consts,
                   inits = my.inits,
                   monitors = my.params,
                   thin = nt, niter = ni, nburnin = nb, nchains = nc)

(outSum1 <- MCMCsummary(out1, round = 2))

What we will do next is plot the posterior distributions for the two variants and the difference.

# extract values from MCs
great.samples <- c(out1$chain1[,'Occ.great'],
                   out1$chain2[,'Occ.great'],
                   out1$chain3[,'Occ.great'])

davyana.samples <- c(out1$chain1[,'Occ.davyana'],
                     out1$chain2[,'Occ.davyana'],
                     out1$chain3[,'Occ.davyana'])

diff.samples <- c(out1$chain1[,'Occ.Diff'],
                  out1$chain2[,'Occ.Diff'],
                  out1$chain3[,'Occ.Diff'])

ggplot() +
  xlim(0,1) +
  geom_histogram(aes(x = great.samples), fill = 'blue') +
  geom_vline(xintercept = outSum1$mean[3]) +
  geom_histogram(aes(x = davyana.samples), fill = 'red') +
  geom_vline(xintercept = outSum1$mean[2]) +
  labs(x = 'Occurrence', y = 'Frequency') +
  annotate(geom = 'text', x = 0.125, y = 350, label = 'greatheadii') +
  annotate(geom = 'text', x = 0.75, y = 300, label = 'davyana')
ggplot() +
  geom_histogram(aes(x = diff.samples), fill = 'grey') +
  geom_vline(xintercept = 0) +
  labs(x = 'Difference in occurence', y = 'Frequency')

Based on the posterior distributions, the davyana variant was detected at more sites than the greatheadii variant. The difference in convincing because most or all of the mass of the posterior distribution for the differences is greater than zero.
