Lesson 10: GLM, overdispersion and offsets

Readings: Kéry (2010) chapter 14

 

Over-dispersion and offsets are two features that are specific to generalized linear models (GLMs) that involve counts. One common type of over-dispersion is one in which there are more zeros than expected; this is called zero-inflation. In this lesson we will address each in the context of the scrub hare example.

 

10.1 Overdispersion

10.1.1 Introduction

For both the Poisson and the binomial distributions, the variability in counts (i.e. the dispersion) is a function of the mean, rather than a separate parameter. For the Poisson, the mean and the variance are both \(\lambda\), and for the binomial, the variance is \(N \times p \times (1 - p)\). As a consequence, for both these parameters, the variance is “built-in” and known if the mean is known. If we fit a GLM to count data, the measure of the variability (the deviance) should be approximately the same value as the residual degrees of freedom, so that their ratio (the mean deviance ratio) is approximately equal to 1.

For count data from the real world, however, it’s common to have counts that are more variable than expected under binomial or Poisson models. We call this over-dispersion, or extra-Poisson or extra-binomial variation, and the residuals are larger than they should be for these distributions. Over-dispersion is common when there are dependencies in our data, such as when animals in groups are treated as independent or important covariates have not been included in the model. The consequence is that confidence intervals will be too narrow and tests more likely to conclude a difference, although it generally won’t affect bias in the means.

In a classical analysis, we can address over-dispersion in the function \({\tt glm()}\) by setting \({\tt family = quasipoisson}\) or \({\tt quasibinomial}\). In BUGS we can specify a distribution like the negative binomial. Also we could add a linear predictor for the Poisson intensity (a normally-distributed random effect). Such a model can be called a Poisson generalized linear mixed model (GLMM) or a Poisson-lognormal model.

 

10.1.2 Data simulation

We repeat our scrub hare data set from lesson 9, but this time we add a normally-distributed site-specific effect in the linear predictor. We will compare to a similar data set without this effect, so you can see what the difference is.

n.site <- 10
x <- gl(n = 2, k = n.site, labels = c('bushveld', 'agriculture'))
eps <- rnorm(2*n.site, mean = 0, sd = 0.5) # normal random effect
lambda.OD <- exp(0.69 +(0.92*(as.numeric(x)-1) + eps) ) # add noise to avg count
lambda.Poisson <- exp(0.69 +(0.92*(as.numeric(x)-1)) ) # for comparison

C.OD <- rpois(n = 2*n.site, lambda = lambda.OD)
C.Poisson <- rpois(n = 2*n.site, lambda = lambda.Poisson)

# boxplots to compare counts
par(mfrow = c(1,2))
ggplot() +
  geom_boxplot(aes(x = x, y = C.OD)) +
  labs(title = 'With OD', xlab = 'Land-use', ylab = 'Hare count') +
  ylim(0, max(C.OD))
ggplot() +
  geom_boxplot(aes(x = x, y = C.Poisson)) +
  labs(title = 'Without OD', xlab = 'Land-use', ylab = 'Hare count') +
  ylim(0, max(C.OD))

 

10.1.3 Classical analysis in R

We’ll start by analysing the over-dispersed data in a GLM, first without correcting, and then by correcting for over-dispersion.

glm.fit.no.OD <- glm(C.OD ~ x, family = poisson)
glm.fit.with.OD <- glm(C.OD ~ x, family = quasipoisson)
summary(glm.fit.no.OD)
summary(glm.fit.with.OD)
anova(glm.fit.no.OD, test = "Chisq")
anova(glm.fit.with.OD, test = "F")

What we see is that the estimates between the two models are the same, but the standard errors and test statistics change considerably.

 

10.1.4 Bayesian analysis with BUGS

To set up the Bayesian analysis, we’ll start with the Poisson t-test from the previous lesson and add over-dispersion to get the Poisson-lognormal model.

# specify the model
model.poisODt <- nimbleCode({
  
  # prior
  alpha ~ dnorm(0, sd = 30)
  beta ~ dnorm(0, sd = 30)
  sigma ~ dunif(0, 10)
  
  # likelihood
  for (i in 1:n) {
    C.OD[i] ~ dpois(lambda[i])
    log(lambda[i]) <- alpha + beta *x[i] + eps[i]
    eps[i] ~ dnorm(0, sd = sigma)
  }
})


# data and constants
my.data <- list(C.OD = C.OD, x = as.numeric(x)-1)
my.consts <- list(n = length(x))

# initial values
my.inits <- function() list(alpha = rlnorm(1), beta = rlnorm(1),
                            sigma = rlnorm(1))

# parameters to estimate
my.params <- c('lambda','alpha', 'beta', 'sigma')

# MCMC settings
nc <- 3
ni <- 3000
nb <- 1000
nt <- 5

# Start Gibbs sampling
# start Gibbs sampler
out1 <- nimbleMCMC(code = model.poisODt,
                   data = my.data,
                   constants = my.consts,
                   inits = my.inits,
                   monitors = my.params,
                   thin = nt, niter = ni, nburnin = nb, nchains = nc)

(outSum1 <- MCMCsummary(out1, round = 2))

To estimate the over-dispersion effects of the added variability, we need to use longer chains. Despite this the run time should be pretty fast.

 

10.2 Zero-inflation

10.2.1 Introduction

We can think of zero-inflation as a specific form of over-dispersion that is commonly found in count data. It means that we have more zeros than we would expect for a Poisson or binomial distribution. One way to interpret this is that some of the survey sites happen to fall in areas that are not habitat for scrub hares (water bodies, bare ground, tarred parking lots, etc), and so the counts there must be zero. In the suitable sites, the counts vary according to the distribution we are using. Regression models that account for these extra zeros are called zero-inflated-Poisson (ZIP) or -binomial (ZIB) models.

A ZIP model looks like this:

\(w_i \sim {\sf Bernoulli}(\psi_i)\)
\(C_i \sim {\sf Poisson}(w_i \times \lambda_i)\)

Interpretation:
\(w_i\): the suitability of a site (1 for yes, 0 for no); a latent or random effect, meaning we cannot observe \(w_i\) perfectly
\(\psi_i\): the probability that a site is suitable
\(\lambda_i\): when \(w_i = 1\), the expected count for site \(i\); when \(w_i = 0\), expected counts are \(0 \times \lambda_i = 0\)

Set up like this, we have a set of two GLMs, one is a logistic regression that describes the suitability of a site, and one is a Poisson regression that describes the variation in counts for suitable sites. Because they are GLMs, they can be expressed as functions of covariates (not necessarily the same ones) on the scale of the link function.

Kéry (2010) makes four comments about ZIP models in the context of animal counts, which I summarise here:

  1. The model allows for two different kinds of zeros:
  1. Bernoulli: zeros come from surveys of unsuitable sites, where no animals are expected to be present;
  2. Poisson: zeros come from surveys of suitable sites which happen to have zero animals in them (i.e. part of the natural variation in a Poisson process).
  1. The ZIP model is a hierarchical model that contains a binary random effect, instead of the normal random effect that we saw previously.
  2. Some advocate widely for the use of ZIP models for count data; however, based on ecology, they appear most suitable in situations where unknown factors affect the suitability of sites. If we have measured factors, they are usually best added as covariates to the Poisson part of the model.
  3. A variant of the ZIP model truncates the Poisson process so that it cannot have zero counts. In principle it might seem easier to have only one type of zero, but biologically it makes more sense for two types because of the possibility of having sites that are suitable but unoccupied.

 

10.2.2 Data simulation

We will generate our data set based on a relatively simple scenario: different densities in bushveld and agriculture and a constant probability of suitable sites (\(\psi\)) in both types of land uses.

# set up the land use indicator variable
n.site <- 20
x <- gl(n = 2, k = n.site, labels = c("bushveld", "agriculture"))
psi <- 0.8 # probability of a suitable site

# establish which sites are suitable
w <- rbinom(n = 2*n.site, size = 1, prob = psi)

# generate expected counts; assumes equal psi between land uses
lambda <- exp(0.69 +(0.92*(as.numeric(x)-1)))

# combine effects of both processes and inspect
C <- rpois(n = 2*n.site, lambda = w *lambda)
cbind(x, w, C) # all unsuitable sites have C = 0

 

10.2.3 Classical analysis in R

We will fit a zero-inflated model using the function \({\tt zeroinfl()}\) in the package pscl.

library(pscl)
fm <- zeroinfl(C ~ x | 1, dist = "poisson")
summary(fm)

The estimates are kinda-sorta close to the input values, but differences are likely because of sampling and estimation error. The “Zero-inflation model coefficient” corresponds to \(1 - \psi\) from the binomial part of the ZIP model, after back-transforming from the link scale.

 

10.2.4 Bayesian analysis with BUGS

The coding in NIMBLE should make it fairly clear which parts of the ZIP model are represented where. For comparison, we also calculate the logit of the zero-inflation parameter from \(\psi\). We will also estimate the latent state \(w_i\), the suitability for hares at each site.

# specify the model
model.ZIP <- nimbleCode({
  
  # priors
  psi ~ dunif(0,1)
  alpha ~ dnorm(0, sd = 30)
  beta ~ dnorm(0, sd = 30)
  
  # likelihood
  for (i in 1:n){
    w[i] ~ dbern(psi)
    C[i] ~ dpois(eff.lambda[i])
    eff.lambda[i] <- w[i]*lambda[i]
    log(lambda[i]) <- alpha + beta *x[i]
  }
  
  # derived quantity
  R.lpsi <- logit(1 - psi)
})

# data and constants
my.data <- list(C = C, x = as.numeric(x)-1)
my.consts <- list(n = length(x))

# Inits function
my.inits <- function() list(alpha = rlnorm(1), beta = rlnorm(1),
                            w = rep(1, 2*n.site))

# parameters to estimate
my.params <- c('lambda','alpha', 'beta', 'w', 'psi', 'R.lpsi')

# MCMC settings (need fairly long chains)
nc <- 3
ni <- 50000
nb <- 10000
nt <- 4

# start Gibbs sampler
out2 <- nimbleMCMC(code = model.ZIP,
                   data = my.data,
                   constants = my.consts,
                   inits = my.inits,
                   monitors = my.params,
                   thin = nt, niter = ni, nburnin = nb, nchains = nc)

(outSum2 <- MCMCsummary(out2, round = 2))

Estimates between the two analysis are fairly similar.

 

10.3 Offsets

10.3.1 Introduction

In a Poisson GLM, we assume that covariates in the model adequately describe expected counts. Sometimes, however, we have a “counting window” that is not constant. For instance, site areas might differ or duration of sampling over time might differ. To account for the variability added by this known factor, we include it as an offset, which allows us to model density (in space or time) rather than the counts directly.

Thinking about our hare data, we have counts that are distributed as a Poisson random variable: \(C_i \sim {\sf Poisson}(\lambda_i)\). If sites have different sizes, then we have \(C_i \sim {\sf Poisson}(A_i \times \lambda_i)\), where \(A_i\) is the size of site \(i\). As a consequence the linear predictor becomes \(\log(A_i \times \lambda_i) = \log(A_i) + \log(\lambda_i)\), and if we want to add a covariate, we have \(\log(A_i \times \lambda_i) = \log(A_i) + \alpha + \beta \times x_i\). Including \(\log(A_i)\) as part of the intercept accounts for the variation caused by differing site area sizes.

 

10.3.2 Data simulation

n.site <- 10
A <- runif(n = 2*n.site, 2,5)       # areas range in size from 2 to 5 km2
x <- gl(n = 2, k = n.site, labels = c('bushveld', 'agriculture'))
linear.predictor <- log(A) + 0.69 +(0.92*(as.numeric(x)-1))
lambda <- exp(linear.predictor)
C <- rpois(n = 2*n.site, lambda = lambda) # add Poisson noise

 

10.3.3 Classical analysis in R

We’ll conduct two analyses using function \({\tt glm()}\), one that ignores offsets and one that includes them.

glm.fit.no.offset <- glm(C ~ x, family = poisson)
glm.fit.with.offset <- glm(C ~ x, family = poisson, offset = log(A))
summary(glm.fit.no.offset)
summary(glm.fit.with.offset)
anova(glm.fit.with.offset, test = "Chisq") # LRT

You might notice that the residual deviance dropped considerably when we included an offset. We can treat this as a sort of correction for over-dispersion caused by variation in area size.

 

10.3.4 Bayesian analysis with BUGS

It’s relatively easy to adapt previous code to give us a model in NIMBLE that includes offsets.

# specify the model
model.offs <- nimbleCode({
  
  # priors
  alpha ~ dnorm(0, sd = 30)
  beta ~ dnorm(0, sd = 30)
  
  # likelihood
  for (i in 1:n) {
    C[i] ~ dpois(lambda[i])
    log(lambda[i]) <- 1 * logA[i] + alpha + beta*x[i]   # Note offset
  }
})

# data and constants
my.data <- list(C = C, x = as.numeric(x)-1, logA = log(A))
my.consts <- list(n = length(x))

# initial values
my.inits <- function() list(alpha = rlnorm(1), beta = rlnorm(1))

# parameters to estimate
my.params <- c('lambda', 'alpha', 'beta')

# MCMC settings
nc <- 3
ni <- 1100
nb <- 100
nt <- 2

# start Gibbs sampler
out3 <- nimbleMCMC(code = model.offs,
                   data = my.data,
                   constants = my.consts,
                   inits = my.inits,
                   monitors = my.params,
                   thin = nt, niter = ni, nburnin = nb, nchains = nc)

(outSum3 <- MCMCsummary(out3, round = 2))
